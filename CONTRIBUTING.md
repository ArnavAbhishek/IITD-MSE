# Contributing Guidelines

Arigatto.. for considering contributing

## How to Contribute

1. **Fork the repository** and create a branch for your contribution.
2. **Add your files** in the correct folder:
   - **Year-wise resources** → `1st-year/`, `2nd-year/`, etc.
   - **Course resources** → `YEAR/SEMESTER/COURSE-CODE/FILE` (e.g., `3rd-year/sem-5/MLL213/minor_17_09_25.pdf`).
   - **Placement experiences** → Add to `placements/comapny_name.md`.
   - **Internship experiences** → Add to `intern/experiences/College_name.md`.
3. **Follow file naming conventions**:
   - Intern and placements resources: add under the relevant markdown file using a new section.
   - Course materials - Add under relevant predefined directories
4. **Commit and push** your changes.
5. **Open a Pull Request (PR)** with a short description of what you added.

## Best Practices

- Keep PRs small and focused.
- Use clear, descriptive file names.
- Update or create a `README.md` in new folders.

## Need Help?

If you are unsure where to add your contribution, open an issue or ask in the PR.
